What it does

Four coaches sharing one rolling chat thread, plus the tools and integrations they call.

  • 🧭 Boss — classifies each turn and routes to the right coach. Falls back to Fitness on any error.
  • 🏋️ Fitness — builds workouts; calls create_workout_session to land them on your calendar.
  • 🥗 Nutrition — meals, recipes, macros. Logs with log_meal.
  • ✅ Productivity — prioritization, goal review, schedule conflicts, and actionable next steps.
  • 🧠 Mental Health — mental-health coach with crisis safety guardrails (Swiss 143 / EU 112). Logs mood snapshots via log_mental_state.
  • 📎 Multimodal attachments — drop images, PDFs, DOCX, or text into chat. Images go to Claude as binary; the rest is text-extracted (capped at 20 000 chars per file).
  • 🎨 Image generation — coaches can call generate_plan_image (OpenAI gpt-image-1); PNG saved to the file store and inlined.
  • 📆 ICS calendar export — per-plan or rolling 30-day feed with Europe/Zurich VTIMEZONE. Drag into your calendar app.
  • 📁 Files — uploads + generated images live on disk at /opt/fitness-agent-data.
  • 🗓️ Auto weekly plan — APScheduler runs Sunday 19:00 Europe/Zurich.
  • ⚙️ Per-user customization — override system prompt and provider per coach; bring your own API keys (encrypted at rest).

How routing decides per turn

  1. You send a message (and optionally attachments).
  2. If the coach is Boss, the classifier (Anthropic by default) picks one of the four sub-coaches based on the message + recent context.
  3. The chosen coach's provider is resolved: your per-user override → .env default → fall back to local if no API key.
  4. If you attached an image and the resolved provider is local (not multimodal), this single turn is bumped to Anthropic so the model actually sees the image.
  5. The agent runs with the resolved system prompt (your override if set, else the built-in default) and tools.
  6. Both turns (yours + the agent's) are written to agentmessage tagged with the resolved coach.

Privacy notice (GDPR)

What's processed, where it goes, how long it's kept, and how to exercise your rights.

Controller. The operator of this instance. For self-hosted personal use, that is you. If you are a user invited to someone else's instance, they are the controller; ask them for their contact details and DPO if applicable.

Lawful basis. Consent and legitimate interest (Art. 6(1)(a) and (f) GDPR) for delivering the coaching service. Health-related metrics (weight, sleep, mood, RPE) fall under Art. 9(2)(a) — explicit consent — which you give by entering them.

Categories of data. Authentication (email + bcrypt password hash), profile (height, weight, goals, equipment, macros), training and meal data, mood / sleep / HRV metrics, chat messages with all coaches, uploaded files (e.g. progress photos), agent-generated images, and per-user settings (system-prompt overrides, provider routing, encrypted API keys).

Recipients (data processors). When you route a coach to a cloud provider, the content of that turn is sent to that provider for inference only:

  • Anthropic, PBC — when a coach is routed to Claude. See Anthropic DPA.
  • OpenAI, L.L.C. — when a coach is routed to GPT, and for image generation. See OpenAI DPA.
  • Local LLM (your own server) — no third-party processing.

International transfers. Anthropic and OpenAI process data in the United States. Both publish Standard Contractual Clauses; using local-LLM routing for any sensitive coach (Mental Health, in particular) keeps that data inside your homelab.

Retention. All data is kept until you delete it. Use "Clear all" in the History tab to wipe chat, "Delete" on each file in the Files tab, or the buttons below to export or remove everything.

Security. API keys are encrypted at rest with Fernet (AES-128-CBC + HMAC-SHA256). Passwords are bcrypt-hashed. Authentication is JWT bearer tokens. Plaintext keys never leave the server. Run the service over HTTPS in any non-loopback deployment.

Your rights (Art. 15–22 GDPR).

  • Access & portability — Settings → "Download my data" emits a ZIP with everything in machine-readable JSON plus your files.
  • Rectification — Settings → Account / Body / Training / Nutrition / Coach prompts.
  • Erasure — Settings → "Delete my account" hard-deletes everything (DB rows + on-disk files).
  • Restriction / objection — switch any coach to local in Settings to stop sending its turns to a third party. The Mental Health coach can run entirely on local LLM.
  • Withdraw consent — same as erasure or restriction; effect is immediate.
  • Lodge a complaint — your national data protection authority (Switzerland: EDÖB; EU: your country's DPA).

Automated decision-making. The coaches generate suggestions; they do not make decisions producing legal effects. You retain full control over what to do with their output.

Delete your account

This is irreversible. Every row you own — profile, chat history, workouts, meals, files — is deleted from the database, and your uploaded/generated files are removed from disk. There is no soft-delete and no recovery. Consider exporting your data first.
